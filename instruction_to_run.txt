SMART FARM IRRIGATION ADVISOR - INSTRUCTIONS TO RUN AND EXPLAIN
================================================================

Project ID: PCSE26-05


1. PROJECT OVERVIEW
-------------------

Smart Farm Irrigation Advisor is a final year project built for smart irrigation decision support. The system helps determine:

- Whether irrigation is required today.
- On which day irrigation may be required in the next 5 days.
- Whether soil moisture is within the safe range.
- Whether excess moisture may create a drainage risk.

The project combines soil sensor data, weather forecast data, a trained AI model, crop-specific rules, and a physics-based water-balance correction layer.

3. HARDWARE EXPLANATION
-----------------------

The hardware layer is responsible for collecting real field conditions.

Expected components:

1. Soil Moisture Sensor
   - Measures the current soil moisture percentage.
   - The backend expects this value in Firebase under the key "soilPercent".

2. Temperature and Humidity Sensor
   - A DHT11, DHT22, or similar sensor can be used.
   - Temperature and humidity help the system understand field conditions and water demand.

3. ESP32, NodeMCU, or Arduino

   - Sends data to Firebase Realtime Database through Wi-Fi.

4. Internet Connection

5. Optional Relay and Water Pump
   - Can be added later for automatic irrigation control.
   - The current implementation provides recommendations; pump automation is part of future scope.

7. AI MODEL ROLE
----------------

The AI model predicts upcoming soil moisture.

Model input features:

- tempmax
- tempmin
- temp
- humidity
- windspeed
- solarradiation
- Evapoatranspiration
- precip
- precipprob
- precipcover
- soil_moisture

Model output:

- Predicted soil moisture percentage

The model gives a raw soil-moisture prediction. Since raw AI predictions can sometimes conflict with real field physics, the project also applies a physics-guided correction layer.


8. PHYSICS-GUIDED CORRECTION
----------------------------

The project uses a simple soil-water balance equation:

next moisture = previous moisture + precipitation * 0.8 - evapotranspiration * 0.5

Meaning:

- Rainfall increases soil moisture.
- Evapotranspiration reduces soil moisture.
- If there is no rainfall, soil moisture should not suddenly increase without reason.

The final prediction combines the raw LSTM output with physics-based water-balance logic. This makes the result more practical, explainable, and suitable for real agricultural decision support.

10. PROJECT SCOPE
-----------------

Current scope:

- Read live soil and weather-related telemetry from Firebase.
- Use a dataset fallback for demonstration when live cloud data is unavailable.
- Fetch current weather and 5-day forecast from OpenWeather.
- Predict upcoming soil moisture using a trained LSTM model.
- Apply physics-guided correction using rainfall and evapotranspiration.
- Generate crop-specific irrigation recommendations.
- Display a 5-day irrigation plan in a Streamlit dashboard.
- Publish recommendation output to Firebase for mobile-app integration.

Out of current scope:

- Fully automatic pump control.
- Large-scale multi-farm deployment.
- Satellite image processing such as NDVI.
- Hardware manufacturing or commercial deployment.

Future scope:

- Add relay-based automatic pump control.
- Add Android app screens that read prediction_output from Firebase.
- Deploy the TFLite model on edge devices.
- Add farm location selection.
- Include satellite-based crop health monitoring.
- Store historical predictions for long-term analytics.


11. HOW TO RUN THE PROJECT
--------------------------

Step 1: Open the project folder.

Command:

cd "D:\Projects\Final Year Project"

Step 2: Go to the backend folder.

Command:

cd SmartFarmBackend

Step 3: Activate the virtual environment.

Command:

.\venv\Scripts\Activate.ps1

If the virtual environment does not exist, create it first:

python -m venv venv
.\venv\Scripts\Activate.ps1

Step 4: Install dependencies.

Command:

pip install -r requirements.txt

Step 5: Set environment variables.

Command:

$env:OPENWEATHER_API_KEY="your_openweather_api_key"
$env:FIREBASE_CREDENTIALS_PATH="predictive-sis-firebase-adminsdk-fbsvc-ad13aadf29.json"
$env:FIREBASE_DATABASE_URL="https://your-project-default-rtdb.asia-southeast1.firebasedatabase.app/"

Important notes:

- Do not commit real API keys or Firebase credentials to a public repository.
- .env.example is only a reference file.

Step 6: Run the Streamlit dashboard.

Command:

streamlit run app.py --server.port 8501

Open this URL in the browser:

http://localhost:8501

16. HOW TO EXPLAIN THE PROJECT DURING EVALUATION
------------------------------------------------

Short explanation:

"This project is built for smart irrigation decision support. It combines soil moisture sensing, temperature and humidity sensing, Firebase cloud storage, weather forecast data, and an LSTM machine learning model to recommend when irrigation should be performed. The project also uses a physics-guided soil-water balance correction layer, which makes the prediction more realistic and explainable. The dashboard shows a 5-day irrigation plan, crop-specific recommendations, model diagnostics, and Firebase publishing support."

Detailed explanation:

"On the hardware side, soil moisture and temperature-humidity sensors collect field data. A microcontroller such as ESP32 sends the readings to Firebase Realtime Database. On the software side, the Python backend reads the live sensor values and fetches 5-day weather forecast data from OpenWeather. The trained LSTM model predicts upcoming soil moisture. The raw prediction is then corrected using rainfall and evapotranspiration-based physics logic. Finally, the selected crop profile threshold is applied to generate the decision: irrigation required, no action, or drainage alert."

18. TROUBLESHOOTING
-------------------

Problem: Streamlit command not found

Solution:

.\venv\Scripts\python.exe -m streamlit run app.py --server.port 8501

Problem: Firebase error

Solution:

- Check the Firebase credential JSON path.
- Check whether FIREBASE_DATABASE_URL is correct.
- Check Firebase Realtime Database rules and service-account permissions.
- The system can still use dataset fallback for demonstration.

Problem: Weather API error

Solution:

- Check whether OPENWEATHER_API_KEY is set.
- Check the internet connection.
- Check whether the API key is active.
- If weather data is unavailable, the backend can use historical dataset fallback.

Problem: TensorFlow, Pandas, or NumPy error

Solution:

- Use the project virtual environment.
- Avoid running the project with a mismatched global Python or Anaconda environment.
- Reinstall dependencies:

pip install -r requirements.txt
